# 기능 카탈로그

NarrativeOS가 제공하는 전체 기능을 카테고리별로 정리한 문서입니다. 엔진 내부 동작은 [아키텍처](ARCHITECTURE.md), API는 [API 레퍼런스](API.md)를 참고하세요.

---

## 1. 엔진 코어

게임을 성립시키는 최소 뼈대입니다.

| 시스템 | 역할 |
|---|---|
| **CampaignState** | 캠페인의 모든 것을 담는 통합 상태 스키마. 필드 소유권 규칙과 버전 마이그레이션 대상 |
| **Narrative Kernel** | 상태를 바꿀 수 있는 유일한 관문. 고정된 요청 타입 집합을 받아 Canon 일관성·무결성을 검증한 뒤에만 반영 |
| **Canon Database** | 인물·장소·사건 등 12타입 엔티티. 바뀌면 안 되는 필드(immutable)와 갱신 가능한 필드를 구분 |
| **Memory Engine** | 5단계 계층(Tier) 기억. 중요도 점수로 인출하고 오래된 것은 압축 |
| **Emotion Engine** | 감정의 파동·피로도(fatigue)·카타르시스 게이팅·공명(Resonance) 모델 |
| **Scene Composer** | 6종 장면 타입(Scene Economy)을 구성하고 여러 디렉터의 지시를 병합 |
| **System Prompt** | 정적 베이스 프롬프트 + 4개 동적 블록 조립, 프롬프트 버전 관리 |

## 2. 서사 심화 시스템

### 세계 & NPC
- **World Simulation** — 화면 밖에서 벌어지는 경량 이벤트를 생성
- **NPC Psychology** — 애착 유형·가치관·방어기제를 가진 심리 모델
- **NPCBrain** — NPC가 스스로 행동을 제안
- **Living NPC** — 오프스크린 상태 갱신
- **NPC Social** — NPC끼리 백그라운드에서 관계를 맺음(큰 변화만 조용히 기억), NPC가 플레이어에게 **먼저 연락**(무시하면 관계 거리가 서서히 벌어짐)
- **NPC Lifecycle** — NPC 아카이빙과 사망 처리 규칙

### 관계
- **Relationship Graph** — 플레이어-NPC뿐 아니라 NPC끼리도 잇는 **10차원** 관계 벡터
- **Relationship Label** — 수치를 정성적 라벨로 변환. 라벨이 전환되면(예: 지인 → 친구) 서사에 반영하고 💞 알림

### 디렉터 (서사 조율)
- **Theme / Rhythm Director**, **Director Debate**, **Drama Manager** — 주제·리듬·긴장을 조율
- **Narrative Director 7단계** — 프롤로그부터 엔딩까지 서사를 구조화
- **Difficulty Director** — 연승/연패를 감지(성공이 이어지면 난이도를 올리되 절대 낮추지 않음)
- **Campaign Planner** — 전체 페이싱 관리

### 메타 엔진
- **Resonance Engine** — 감정 공명(완전판)
- **Self Reflection** — 후처리 추출 호출에 병합(호출 수 절감)
- **Campaign Health** — 캠페인 건강도 진단, Narrative DNA
- **AI Watchdog & Hallucination Checker** — 설정 붕괴 감지(심각하면 서사를 조용히 한 번 재생성), 죽은 인물 재등장 차단, 미등록 고유명사를 Canon 등록 후보로 수집

### 스토리 요소
- **Legacy Engine** — 세대교체(주인공 사망/은퇴 시 다음 세대로)
- **Dynamic Quest** — 상황에 맞춰 퀘스트 생성
- **Consequence Chains** — 선택의 결과가 이어지는 연쇄를 추적
- **Mystery** — 단서 수집·추적
- **Hidden Variables** — 이성(sanity)·피로·타락 등 9종의 숨은 변수. 이 중 다수가 **행동 경향**으로 서사에 반영됨(설명문이 아니라 뉘앙스로)
- **동적 특성 엔진** — 사건에 반응해 특성이 자동으로 생겨나고(예: 임신·부상·배신 후) 실시간으로 증감하며 오래 방치되면 사라짐. 장르별 기본 스탯 프리셋(예: 마력, 전기 이해도)과 플레이어 수동 추가도 지원

## 3. 게임플레이 시스템

- **Dice / SkillCheck** — 주사위 판정, Difficulty Director와 연동
- **Ending System** — 조건 기반 분기 엔딩
- **Inventory** — 아이템과 서사 판정 기반 조합. `추억(sentimental)` 태그 아이템은 획득 당시 기억을 소환하는 특수 서술
- **선택 가중치 힌트** — 스탯과 대조해 유리/보통/불리 배지 표시(수치는 노출하지 않음)
- **Rumor & Faction** — 소문 생성·전파, 세력 평판 변화
- **시대별 통신** — 세계의 기술 수준(`tech_level`)에 따라 채널이 달라짐. 고대는 **편지**(거리에 따른 배달 시간, 관계·충돌에 따른 가로채기 확률, 가로채이면 소문 누출), 현대는 DM·게시판. 비동기 배달 큐로 처리
- **이벤트 카운트다운** — 플레이어에게 공개된 사건만 남은 시간을 표시(스포일러 방지)
- **선택 히스토리 트리** — 중요한 분기 선택을 기록해 "여정" 타임라인으로 시각화

## 4. 캠페인 생명주기

- **세이브 마이그레이션** — 스키마 버전을 순차 적용, 로드 실패 시 백업(`.bak`) 보존
- **캠페인 아카이브** — 진행중/완결/전체 필터로 런처에서 구분
- **세계관 템플릿** — 세계관을 저장해 다른 캠페인에서 재사용(앤솔로지 모드로 Canon 공유)
- **사망/은퇴 확인 게이트** — 주인공의 죽음·은퇴는 자동 확정하지 않고, 사람이 확인해야만 다음 세대로 넘어감
- **API 키 풀 로테이션** — 여러 키를 등록해 한도 소진 시 자동 전환(키 값은 노출하지 않음)
- **콘텐츠 가드레일** — 캐릭터 생성 시 미성년+로맨스 조합 등 부적절한 설정을 차단

## 5. UI

프레임워크 없는 순수 JS/CSS 단일 페이지 앱입니다.

### 런처 & 마법사
- **런처** — 캠페인 카드 목록, 진행중/완결 필터, 빈 상태 안내, 삭제, JSON 가져오기
- **설정 마법사** — 세계관·캐릭터를 3단계로 생성(입력 폼 우선), 빠른 시작, AI 실패 처리, 임시저장(draft), 템플릿으로 시작

### 상단 탭바 6개
[README](../README.md#-화면-구성)의 표 참고. 이야기 탭은 독백(괄호)을 별도 스타일로, 마크다운 강조와 감정에 따른 타이핑 속도를 지원합니다.

### 편의기능 (약 38종)
입력/상호작용, 열람/탐색, 저장/이력, 개인화, 소소한 재미, 생산성, 접근성 범주로 나뉘며 대부분 프론트엔드 단독으로 동작합니다. "사람이 직접 트리거하고 AI는 보조" 원칙을 따릅니다.

### Advanced 패널 (개발자용, 기본 꺼짐)
감정·심리·관계·기억·Canon·스토리 구조·난이도·세계·단서·건강도·디렉터 로그·프롬프트 뷰어·성능 프로파일 등 11개 이상의 서브탭과 전체 검색을 제공합니다.

## 6. 확장: 테마 & 플러그인

> AI가 생성한 실행 코드를 앱에 주입하지 않습니다. AI는 정해진 스키마의 값만 채우고, 실행 로직은 전부 미리 만들어둔 코드가 담당합니다.

### 커스텀 테마
자유서술로 원하는 분위기를 설명하면 AI가 CSS 토큰(색상·폰트·radius)을 채웁니다. **검증**을 반드시 거칩니다 — 변수명 화이트리스트, 색상/폰트/수치 형식 검사, `url()`·`@import`·`javascript:` 등 외부 로드·스크립트 스킴은 무조건 거부. 미리보기 후 확정하며, 앱 전역에 저장됩니다.

### 플러그인 (선언적 매니페스트)
아래 **5개 확장 지점**에만 값을 추가할 수 있습니다.

| 확장 지점 | 추가할 수 있는 것 |
|---|---|
| 장면 타입 | 새 장면 태그(렌더링은 기존 로직 재사용, 이름·톤 문구만 새로) |
| 장르 스탯 프리셋 | 새 장르-스탯 매핑(예: `사이버펑크 → 해킹 숙련도`) |
| 통신 채널 | 새 기술 수준에 대응하는 채널(배달 로직은 기존 재사용, 겉모습만) |
| 하우스룰 번들 | 이름 붙인 자유 규칙 묶음 |
| Advanced 위젯 | 기존 데이터를 다르게 배치하는 위젯(새 데이터 소스는 불가) |

이 외의 타입·필수 필드 누락·잘못된 데이터 소스는 거부됩니다. 새 Director, Kernel 요청 타입, 상태 스키마 변경은 플러그인 범위 밖입니다. 미리보기(사람이 읽는 요약) → 확정 → 앱 전역 등록 흐름을 따르며, 설정 탭에서 켜고 끌 수 있습니다.

## 7. 성능 & 인프라

플레이어에게 직접 보이진 않지만 비용과 안정성을 지탱하는 기능입니다.

- **컨텍스트 캐시** — 정적 프롬프트 블록을 해시 캐시하고, 하우스룰·강도·프롬프트 버전이 바뀌면 무효화
- **델타 컨텍스트 & 동적 LOD** — 바뀌지 않은 Canon/기억은 다시 전송하지 않고, 참가자별로 정보 상세도를 차등
- **토큰 예산 관리** — 블록별 예산을 초과하면 우선순위 낮은 항목부터 제거
- **결정론적 검증기** — AI 응답의 JSON 파싱을 3단계로 복구, 계속 실패하면 무결성 로그에 경고
- **증분 상태 & 스냅샷** — 변경된 키를 저널링하고, 100턴마다 gzip 스냅샷(최근 3개 보관)
- **통합 스케줄러** — 모든 백그라운드 주기 작업을 단일 설정으로 관리
- **시간 가속** — 30일 이상 건너뛰면 오프스크린 배치로 처리하고 "그동안 있었던 일" 카드로 요약
- **저토큰 모드** — 선택적 호출(인터넷 검색 생성, NPC 배경 상호작용 등)을 끄거나 빈도를 줄여 핵심 2콜만 유지
- **관측 도구** — 프롬프트 원문 뷰어, 단계별 성능 프로파일러, Explain 모드, 최근 호출 기록
